# Context Sink Process Reconstruction

The Context Sink spec-building process is the primary exemplar for SpecOps.

Observed stages:

```text
1. initial Teams/meeting tagging idea
2. category/folder/manifest model
3. raw sink reframing
4. LLM tags and many-to-many context
5. graph locality / GraphRAG exploration
6. permission laundering concern
7. ingestion modes and delegated auth
8. source-specific feasibility
9. context dump #1
10. context dump #2
11. adversarial relay stress test
12. derived unified spec
13. subsystem recursive deepening
14. spec repository emission
```

Reusable primitives:

- Reframe product model.
- Extract design axes.
- Identify central risk.
- Run adversarial hardening.
- Convert hardening insights into architecture primitives.
- Produce derived spec.
- Deepen subsystems recursively.
- Emit repository.
- Evaluate reproduction.

SpecOps must encode these primitives as commands and eval rubrics rather than a single giant prompt.
